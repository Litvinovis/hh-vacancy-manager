# HH Vacancy Manager

Многопользовательский трекер вакансий с HH.ru: сбор через RSS (или по сохранённой ссылке с фильтрами hh.ru), полный текст — через headless-браузер, оценка релевантности — через LLM, отдельный личный кабинет и набор поисков на каждого пользователя.

## Как это работает

```
RSS / ссылка hh.ru (discovery) → новый hh_id?
  → дешёвый AI-прескрининг карточки (без скрейпинга)
  → scraper sidecar (Playwright, реальный браузер) → полная карточка вакансии
  → AI-анализ (LLM, критерии конкретного поиска) → Telegram-уведомление
```

RSS не блокируется ботозащитой hh.ru, но не содержит текста вакансии и ограничен 20 результатами без пагинации — поэтому есть альтернатива: поиск по сохранённой ссылке (собранной вручную в браузере со всеми фильтрами hh.ru), дающей настоящую пагинацию и более гибкий отбор. Полный текст в обоих случаях достаёт `scraper/` — отдельный Node.js/Playwright-сервис с одним тёплым Chromium, обходящий блокировку DDoS-Guard так же, как обычный браузер.

Дубли одной и той же вакансии (клоны по городам, повторные публикации) распознаются и не гоняются через AI повторно — вердикт переиспользуется. Уже одобренные вакансии периодически перепроверяются на актуальность (сняты ли с публикации). Список AI-моделей поддерживает себя сам: живые free-модели проверяются по расписанию, а при исчезновении одной из них замена подбирается тем же LLM.

## Пользователи и доступ

Каждый пользователь — своя роль (`user`/`admin`), свой логин и до 3 независимых поисков. Общие (`global`) поиски создаёт только admin — их результат виден всем, статус (отклик/отклонено) при этом свой у каждого. Обычный пользователь видит и запускает только свои данные; admin — всё.

## Стек

- **Backend:** Java 25, Spring Boot 4.1, SQLite (JDBC, без ORM)
- **Scraper sidecar:** Node.js, Playwright (headless Chromium)
- **Frontend:** HTML/CSS/JS (vanilla, без сборки и фреймворков)
- **AI:** любой OpenAI-совместимый chat-completions API, настраивается в админке, с fallback-цепочкой провайдеров

## Быстрый старт

Требуется Java 25+, Maven 3.8+, Node.js 24+.

```bash
# Backend
cd backend
mvn package -DskipTests
java -jar target/hh-gui-1.0.0-SNAPSHOT.jar
# или: mvn spring-boot:run

# Scraper sidecar (отдельный процесс, обязателен для полноценного сбора)
cd scraper
npm ci
npx playwright install --with-deps chromium
node server.js
```

Backend — `http://localhost:8080`, scraper sidecar — `http://localhost:8095`. При первом запуске на пустой БД в логе появится сгенерированный пароль admin.

```bash
cd backend && mvn test
```

## Деплой

Self-hosted GitHub Actions runner собирает `backend/` и `scraper/`, раскладывает по `/opt/hh-gui` и перезапускает systemd-юниты `hh-gui`/`hh-scraper` (см. `.github/workflows/deploy.yml`, юниты — в `scripts/`).
